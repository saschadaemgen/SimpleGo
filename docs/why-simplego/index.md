# Why SimpleGo?

SimpleGo exists because secure messaging shouldn't require a smartphone, a phone number, or trust in a centralized ecosystem.

## The Case for SimpleX

SimpleX is the only messaging protocol designed from the ground up with **no user identifiers**. No phone numbers, no usernames, no keys on servers. Each connection is an independent pair of anonymous queues.

## The Case for Dedicated Hardware

Smartphones are surveillance devices by design — cellular tracking, app store control, OS-level telemetry. A dedicated messaging device eliminates these attack vectors entirely.

## The Case for a Native Implementation

The official SimpleX client is written in Haskell — a language that requires a full runtime environment and garbage collector. SimpleGo proves the protocol can run natively in C on a $5 microcontroller with 320KB of RAM.
